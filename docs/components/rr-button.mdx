## Overview

Use `rr-button` for any action the user takes — submit, cancel, delete, or an action that isn't navigation. It carries full keyboard and form semantics (Enter/Space activate) plus a built-in `loading` state. Use `variant="primary"` for the single most important action on a view, `secondary` (outlined) for supporting actions, `ghost` (text-only, no border) for the quietest actions — dismiss, tertiary options, dense toolbars — and `danger` for destructive ones. For something that navigates to another page, use an anchor (`<a>`), not a button.

**Form participation:** the element is form-associated (`ElementInternals`), so `<rr-button type="submit">` inside a `<form>` submits it exactly like a native submit button; `type="reset"` resets. No wrapper or event plumbing needed.

## Accessibility

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/button/](https://www.w3.org/WAI/ARIA/apg/patterns/button/)
- **Focusable:** yes
- **Keyboard:** Enter activates the button; Space activates the button; Tab moves focus to/from the button
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 2.4.6 Headings and Labels, 4.1.2 Name, Role, Value

**Rules**

- delegatesFocus ensures host element forwards focus to inner <button>.
- aria-busy='true' set during loading state to communicate to AT.
- Icon-only buttons require aria-label for accessible name.
- disabled attribute removes from tab order and prevents activation.
- Click events are suppressed when disabled or loading.

## Usage

- **Do** limit `variant="primary"` to one button per page or view — if everything is primary, nothing is.
- **Do** give icon-only buttons an `aria-label` — the icon has no accessible name on its own.
- **Do** use `loading` for in-flight actions; it sets `aria-busy` and suppresses repeat clicks.
- **Don't** use a button to navigate to another page — that's a link's job (`<a>`), which supports open-in-new-tab and is announced correctly.
- **Don't** make `disabled` the only feedback for a blocked action — pair it with helper or error text explaining why.
