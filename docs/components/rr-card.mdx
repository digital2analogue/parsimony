## Overview

Use `rr-card` to group related content on its own surface — a settings panel, a summary block, a list item with chrome. It follows the system's flat-elevation rule: separation comes from `background.alt` and a quiet `border.alt` frame, not shadows. Header, body, and footer are optional slots; empty header/footer rows collapse away. The `padding` prop (`none | sm | md | lg`) controls body padding only — header and footer padding is fixed.

**Slotted heading typography:** the header row itself supplies the type style (`--font-label-strong-medium`), which slotted plain text or a `<span>` inherits automatically. If you slot a real heading element for document-outline reasons (`<h2 slot="header">`), the page's global heading styles will override that inheritance — neutralize them (`font: inherit; margin: 0`) so the card's header type wins.

## Examples

### Body only

```html
<rr-card>Simple card content.</rr-card>
```

### Header, body, footer

```html
<rr-card>
  <h2 slot="header">Card title</h2>
  <p>Card body text.</p>
  <span slot="footer">Last updated today</span>
</rr-card>
```

### Large padding

```html
<rr-card padding="lg">Roomy card.</rr-card>
```

## Properties

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `padding` | `'none' \| 'sm' \| 'md' \| 'lg'` | `md` | Internal padding around the body slot. Header and footer padding is always fixed at `tight/element`. |

## Slots & Events

### Slots

| Slot | Description |
|------|-------------|
| `header` | Card heading row (title, actions, avatar). Hidden when empty. |
| `default` | Card body content. |
| `footer` | Card footer row (secondary actions, metadata). Hidden when empty. |
