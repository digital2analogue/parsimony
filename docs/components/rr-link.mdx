## Overview

Use `rr-link` for navigation — anything that takes the user somewhere. For an action that changes state, use `rr-button` (ghost variant if it must look quiet). It enforces the system's inverted link convention: underlined at rest, underline removed on hover, phosphor-green `foreground-action` text. `variant="muted"`/`"subtle"` step the color down for dense meta contexts while keeping the underline affordance. Because the convention is inverted from the web default, never hand-style anchors in consumer code — wrap them in `rr-link` (or copy the documented global CSS) so hover behavior stays consistent.

## Properties

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `href` | `string` | `` | Link destination. |
| `target` | `'_self' \| '_blank' \| '_parent' \| '_top'` | `_self` | Browsing context. Use "_blank" to open in a new tab. |
| `rel` | `string` | `` | rel attribute. Defaults to "noopener noreferrer" when target="_blank" unless overridden. |
| `variant` | `'default' \| 'muted' \| 'subtle'` | `default` | Colour variant. Reflected to an attribute. |
| `aria-label` | `string \| null` | `null` | Accessible label forwarded to the anchor when link text alone is insufficient. |

## Accessibility

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/link/](https://www.w3.org/WAI/ARIA/apg/patterns/link/)
- **Focusable:** yes
- **Keyboard:** Enter activates the link; Tab moves focus to and from the link
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 2.4.4 Link Purpose (In Context), 2.4.7 Focus Visible, 4.1.2 Name, Role, Value

**Rules**

- rel="noopener noreferrer" is applied automatically for target="_blank" to prevent reverse tabnabbing
- Provide aria-label when the visible link text is not descriptive on its own
