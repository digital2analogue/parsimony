## Overview

`rr-menu-item` is a single command inside `rr-menu` — it exists only in that context, the way `rr-tab` exists only inside `rr-tab-list`. The host element itself carries `role="menuitem"` and a managed `tabindex`, so the flattened accessibility tree reads menu → menuitem without any wrapper gymnastics; focus arrives via the parent menu's keyboard navigation, never via Tab.

Two modifiers cover the real-world cases: `danger` for destructive actions (danger foreground, red-tinted hover — reserve it for genuinely destructive things), and `disabled` for temporarily unavailable actions (skipped by keyboard nav, inert to clicks, but still visible — a hidden option is an undiscoverable option).

Activating an item bubbles `rr-menu-item-select` up to the menu, which closes, returns focus to the trigger, and re-emits `rr-menu-select` — consumers listen on the menu, not on individual items.

## Accessibility

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/menu-button/](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/)
- **Focusable:** yes
- **Keyboard:** Enter or Space activates the item; Focus arrives via the parent menu's roving keyboard navigation
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 4.1.2 Name, Role, Value

**Rules**

- role=menuitem and tabindex=-1 live on the host so the flattened tree reads menu → menuitem
- Disabled items expose aria-disabled=true and stay visible — hidden options are undiscoverable options

## Usage

**Do** give every item a `value` — it's what arrives in `rr-menu-select`; an item without one reports an empty string and forces label-string matching in the handler.

**Do** write labels as verb phrases ("Edit rule", "Duplicate", "Delete rule") — a menu is a list of commands, not a list of nouns.

**Do** let the label say what's destroyed on `danger` items ("Delete rule", not just "Delete") — the red is a signal, not the message.

**Don't** nest interactive elements inside an item (no buttons, links, or toggles in the slot) — the item itself is the interactive unit; nested controls break the menuitem contract and keyboard activation.

**Don't** use `disabled` to hide capabilities permanently — disabled communicates "not right now". If an action never applies to this object, omit the item.
