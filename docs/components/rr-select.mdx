## Overview

Use `rr-select` for choosing exactly one option from a known list — it wraps a real native `<select>`, so keyboard and screen-reader behavior come from the platform. For free-text entry use `rr-input`; for on/off use `rr-toggle`; for a short mutually-exclusive set you want fully visible, use `rr-radio-group`.

**Options are a JS property, not markup.** There is no `<option>` slot — assign an array of `{ value, label, disabled? }` to the `options` property:

```html
<rr-select id="theme" label="Theme" name="theme" value="dark"></rr-select>
<script>
  document.getElementById('theme').options = [
    { value: 'system', label: 'System' },
    { value: 'dark', label: 'Dark' },
    { value: 'light', label: 'Light' },
  ];
</script>
```

Ordering is safe: Lit captures properties assigned before the element upgrades, and `value` is (re-)applied to the native select whenever `value` or `options` change — so an initial `value` attribute correctly selects its option even when `options` arrive later. If `value` matches no option, the native select keeps its own selection. Wrapping the assignment in `customElements.whenDefined('rr-select')` is harmless but not required.

## Properties

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | `` | Form field name submitted with the selected value. |
| `value` | `string` | `` | Selected option value. |
| `label` | `string` | `` | Field label rendered above the select. |
| `helperText` | `string` | `` | Helper text shown when there is no error. Maps to attribute helper-text. |
| `errorText` | `string` | `` | Error message; sets data-invalid and a custom validity. Maps to attribute error-text. |
| `required` | `boolean` | `false` | Marks the field as required and shows an asterisk. |
| `disabled` | `boolean` | `false` | Disables the select. |
| `options` | `SelectOption[]` | `[]` | Options to render: array of { value: string; label: string; disabled?: boolean }. |
