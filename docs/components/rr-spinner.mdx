## Overview

Use `rr-spinner` for short, unquantified waits — a submit in flight, a panel refreshing. It announces itself to screen readers (status semantics with an accessible `label`, "Loading" by default) so don't stack extra live regions around it. Choose by context: skeleton for first-load layout placeholders, progress bar when you can measure completion, spinner for everything brief. Inside `rr-button`, prefer the button's own `loading` prop over slotting a spinner — it also suppresses clicks and sets `aria-busy`. The rotation honors `prefers-reduced-motion` (a frozen arc plus the label still communicates busy).
