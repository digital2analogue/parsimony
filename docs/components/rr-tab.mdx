## Overview

`rr-tab` is a single tab button inside `rr-tab-list` — it declares its `value` and label, and the parent list manages selection, keyboard order, and events. Don't use it standalone: alone it has no tablist semantics and its roving tabindex never activates. Keep labels to one or two words; if a tab needs a paragraph to explain itself, the views probably aren't peers.

## Accessibility

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/tabs/](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/)
- **Focusable:** yes
- **Keyboard:** Enter or Space activates the focused tab via its native button
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 2.4.7 Focus Visible, 4.1.2 Name, Role, Value

**Rules**

- Roving tabindex (selected tab is 0, others -1) and aria-selected are managed in concert with rr-tab-list.
- Host uses delegatesFocus so focus forwards to the inner button.
