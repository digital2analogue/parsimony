## Overview

`rr-table-cell` is a cell inside `rr-table-row`. The host carries `role="cell"` — flipping to `role="columnheader"` when `header` is set — and lays out as a CSS table cell, with padding inherited from the parent table's `density`. The slot takes arbitrary content: text, an `rr-badge`, an `rr-menu` overflow trigger, an amount.

Two modifiers cover the column archetypes: `header` renders the system's column-header recipe (muted, uppercase, all-caps tracking, label-strong-small — headers recede so data leads), and `numeric` right-aligns in JetBrains Mono with tabular figures so digit columns stack cleanly.

## Usage

**Do** set `numeric` on the header cell *and* the body cells of a numeric column — alignment is per-cell, and a left-aligned header over right-aligned digits looks off.

**Do** keep cell content to one line where you can; tables scan by row, and tall wrapped cells break the rhythm.

**Don't** use `numeric` for text that merely contains digits (dates, phone numbers, IDs with meaning) — it's for quantities that benefit from digit alignment.

**Don't** nest another table inside a cell. If a cell needs that much structure, the row wants a detail view (expandable row, dialog) instead.
