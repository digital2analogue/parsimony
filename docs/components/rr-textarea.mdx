## Overview

Use `rr-textarea` for multi-line free text — descriptions, comments, messages. It mirrors `rr-input`'s contract exactly (label, helper/error text, `required`, ElementInternals form participation), plus `rows` for initial height, so a form mixing the two stays uniform. The same field recipe applies: `error-text` drives the error state and `aria-invalid`/`role="alert"` wiring; helper text and error text share the described-by slot. If input is single-line by nature (names, emails, URLs), use `rr-input` — a textarea invites prose.

## Accessibility

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/textbox/](https://www.w3.org/WAI/ARIA/apg/patterns/textbox/)
- **Focusable:** yes
- **Keyboard:** Standard textarea editing keys apply to the inner native textarea
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 2.4.7 Focus Visible, 3.3.1 Error Identification, 3.3.2 Labels or Instructions, 4.1.2 Name, Role, Value

**Rules**

- Host uses delegatesFocus so focus forwards to the inner native textarea.
- Error state sets aria-invalid and aria-errormessage; helper/error are wired via aria-describedby. The error container has role=alert.
- Form-associated via ElementInternals (formAssociated = true); the value participates in native form submission.
