## Overview

Use `rr-toggle` for a binary setting that **applies immediately** — notifications on/off, dark mode, feature enables. If the choice only takes effect on form submit, use `rr-checkbox` instead: the switch shape promises instant effect, and breaking that promise erodes trust in every other toggle. Label it with the slot (or `label` prop) as a noun-phrase state ("Email notifications"), never a question, and never make the label change with the state — the switch position communicates on/off. It's form-associated (`name`/`value` submit like a checkbox) and the thumb transition respects reduced motion.

- **WAI-ARIA pattern:** [www.w3.org/WAI/ARIA/apg/patterns/switch/](https://www.w3.org/WAI/ARIA/apg/patterns/switch/)
- **Focusable:** yes
- **Keyboard:** Space toggles the switch; Enter toggles the switch
- **WCAG:** 1.4.3 Contrast (Minimum), 2.1.1 Keyboard, 2.4.7 Focus Visible, 4.1.2 Name, Role, Value

**Rules**

- A visually hidden native checkbox with role=switch provides focus, keyboard, and label-click semantics; aria-checked reflects the state.
- Form-associated via ElementInternals (formAssociated = true); the value participates in native form submission when checked.

## Usage

- **Do** apply the change the moment the toggle flips — persist it and reflect failure by flipping back with an `rr-alert`, not by silently ignoring the click.
- **Do** keep the label a stable noun phrase ("Email notifications") — the switch position says on/off; a changing label ("Enable"/"Disable") makes the current state ambiguous.
- **Do** place the label left of the switch in settings rows (see the Assembly patterns in AGENTS.md).
- **Don't** use a toggle inside a form that has a Save button — that's a checkbox's contract.
- **Don't** use it for anything but binary state; three-way choices are a radio group or select.
