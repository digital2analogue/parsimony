Architecture notes

Deleted tokens · Naming rules · Known drifts
Naming convention — hard rule
⚑
Accent tokens must always include the color name. Text color: foreground-accent-blue, foreground-accent-green, foreground-accent-purple. Inverse (text on a filled accent surface): foreground-accent-on-blue, foreground-accent-on-green, foreground-accent-on-purple. Wrong: foreground-accent or foreground-on-accent-blue — bare or inverted namespace, deleted 2026-04-28.
Tokens deleted 2026-04-27
TokenReason for deletion
foreground-accentGeneric — replaced by named foreground-accent-[color] slots. No stable semantic meaning.
background-accentRedundant with background-action in DE. Two tokens, same value, no distinction.
foreground-on-accentPaired counterpart to deleted background-accent.
foreground-primaryRedundant with foreground-default. Identical value, no distinction.
feedback-errorRedundant with foreground-danger — both were #C8002E. Two tokens, same value.
feedback-danger-foregroundRenamed to foreground-danger. Removed feedback namespace — foreground/background structure handles all roles.
foreground-accent-redRenamed to foreground-danger. Red was the only non-green accent in the old palette — consolidated into the danger semantic role.
feedback-danger-background-subtleRenamed to background-danger-alt. Removed feedback namespace.
