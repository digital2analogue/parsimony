Color primitives

Raw hex values — the single source of truth for every color in the system. Never reference these in UI code; use semantic tokens instead. Numeric scale: low = light, high = dark. Named slots (sky, navy, chip, approve, etc.) are used where a value cannot fit a monotonic numeric ramp without inversion.

Motion

tokens/semantic/motion.tokens.json
↓
Motion in this system is understated — terminal aesthetic reads as purposeful and calm. Use the minimum duration that communicates the state change. Hover a duration row to preview it.
Duration
--motion-duration-instant 120ms
Hover states, focus rings, color swaps. Felt but not watched — the interaction feels immediate.
--motion-duration-standard 200ms
Dropdowns, tooltips, expand/collapse. Snappy and legible — the workhorse of the system.
--motion-duration-emphasized 350ms
Modals, page transitions, large layout changes. The eye tracks this consciously. Never on hover states.
Easing
--motion-easing-enter cubic-bezier(0, 0, 0.58, 1)
Elements entering the screen. Starts fast, decelerates to rest — signals arrival. Most common.
--motion-easing-exit cubic-bezier(0.42, 0, 1, 1)
Elements exiting the screen. Starts slow, accelerates to departure — signals leaving.
--motion-easing-move cubic-bezier(0.42, 0, 0.58, 1)
Elements repositioning within the viewport — sliding panels, tab indicators, loading bars.
--motion-easing-default cubic-bezier(0.25, 0.1, 0.25, 1)
General-purpose. Color and opacity transitions with no directional intent. Falls back gracefully.
